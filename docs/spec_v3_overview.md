# タビモン V3 仕様書

> バージョン: V3（初回リリース）
> 更新日: 2026-03-29
> 対象: `tabimon-browser`

## 1. この文書の位置づけ

本書は、タビモン V3 初回リリースの企画・体験・画面仕様をまとめた正本です。  
技術仕様は [spec_v3_technical.md](./spec_v3_technical.md) を参照してください。

V2 の土台仕様は [spec_v2_overview.md](./spec_v2_overview.md) と [spec_v2_technical.md](./spec_v2_technical.md) を前提とし、本書では V3 で追加・変更した内容のみを定義します。

## 2. V3 の定義

V3 は、V2 の「相棒が主役の UI」を維持したまま、育成の手応えを強化するアップデートです。  
テーマは「相棒との関係を深くする」です。

## 3. V3 の目的

- 毎回の冒険で成長実感を持てるようにする
- レベル以外の育成軸を追加して、遊びの密度を上げる
- 物語を進める遊びと、育成周回の遊びを分けて遊びやすくする
- 相棒を主役として見せる導線を、数値面でも意味のあるものにする

## 4. 今回の実装対象

V3 初回リリースで対象とするのは以下です。

- EXP 表示強化
- きずなシステム
- スキル熟練システム
- ステージ選択の `ものがたり / そだてる` 分離
- ホーム画面改修
- 仲間詳細画面改修
- ステージ選択画面改修
- リザルト画面改修
- 軽量進化演出
- V2 → V3 データマイグレーション

## 5. 今回の対象外

V3 初回リリースでは以下を含めません。

- 分岐進化の本実装
- きずな進化
- スキル選択分岐
- 本格ストーリー多話化
- 図鑑の本格改修
- 設定画面の新設・改修
- BGM / SE の強化
- PWA 強化
- `/evolution` 新規ページ
- 大量アニメーション
- 大量アセット追加

## 6. 主要追加体験

### 6.1 EXP 表示強化

- ホームで相棒の EXP 要約を表示する
- 仲間詳細で EXP を確認できる
- リザルトで今回増えた EXP が分かる
- 初回クリアボーナス EXP を表示する

### 6.2 きずなシステム

- 相棒に `bondPoints` を持たせる
- `bondPoints` から `bondRank` を決定する
- ホームで相棒タップ時に 1 日 3 回まで `+1`
- 冒険完了時に結果に応じてきずなを加算する
- リザルトで今回の増加量とランク変化を表示する

### 6.3 スキル熟練

- 相棒スキルの使用回数を記録する
- 使用回数に応じて熟練段階を更新する
- リザルトで今回成長したスキルを表示する
- 仲間詳細で各スキルの熟練を確認できる

## 7. 数値仕様

### 7.1 きずな段階

| ランク | 必要 pt |
|---|---:|
| 0 | 0 |
| 1 | 50 |
| 2 | 150 |
| 3 | 400 |
| 4 | 1000 |

### 7.2 きずな加算

| 行動 | 加算 |
|---|---:|
| ホームタップ | +1 |
| ホームタップ 1 日上限 | 3 回 |
| 冒険成功 | 定数加算 |
| 冒険失敗 | 定数加算 |
| リタイア | 定数加算 |

※ 冒険時の具体値は技術仕様書と定数定義を正本とします。

### 7.3 熟練段階

| 段階 | 必要回数 |
|---|---:|
| 0 | 0 |
| 1 | 5 |
| 2 | 15 |
| 3 | 30 |

## 8. ステージ選択の変更

`/adventure/stages` は、V3 で以下の 2 タブ構成になります。

- `ものがたり`
- `そだてる`

### 8.1 ものがたり

- 既存ステージを物語導線として表示する
- 初回クリアボーナス EXP を見せる
- 解放順に進める構造を維持する

### 8.2 そだてる

`そだてる` は、以下の 3 カテゴリ × 前半 / 後半 の 6 ステージ構成とする。

- `EXP`
- `BOND`
- `SKILL`

#### 6 ステージ構成

| カテゴリ | ステージ名 | 想定帯 |
| --- | --- | --- |
| EXP | けいけんちの草原・前半 | Lv1〜14 |
| EXP | けいけんちの草原・後半 | Lv15〜30 |
| BOND | きずなの広場・前半 | Rank0〜2 |
| BOND | きずなの広場・後半 | Rank2〜4 |
| SKILL | わざみがきの丘・前半 | Stage0〜2 |
| SKILL | わざみがきの丘・後半 | Stage2〜3 |

補足:

- `しんか素材 / なかまさがし` は V3.5 以降の検討対象へ移す
- 初回 V3 の `そだてる` は、レベル、きずな、わざみがきの 3 軸に絞る

## 9. 画面ごとの変更

### 9.1 ホーム `/home`

- 相棒を主役として表示
- 相棒の `EXP / きずな` 要約を表示
- 相棒カード全体から詳細へ遷移できる
- 相棒タップで軽いリアクション兼 `きずな +1`

### 9.2 仲間詳細 `/monsters/[id]`

- `EXP / きずな / 熟練` の 3 軸を表示
- 既存責務である相棒設定・手放しは維持

### 9.3 ステージ選択 `/adventure/stages`

- `ものがたり / そだてる` の 2 タブ
- `STORY` と `FARM` でカードの見せ方を分離

### 9.4 リザルト `/adventure/result`

- EXP セクション
- きずなセクション
- 熟練セクション
- 進化通知または軽量進化演出
- CTA

### 9.5 進化演出

- 新規ページ追加はしない
- リザルト内で短く気持ちよく見せる
- 重いアニメーションは避ける

## 10. データ移行

V2 セーブから V3 セーブへ安全に移行する。

移行ルール:

- `bondPoints = 0`
- `bondRank = 0`
- 全スキル熟練は `useCount = 0`, `stage = 0`
- 既に進化済み個体は維持
- 図鑑進行は維持
- 不足項目は安全な初期値で補完
- 一度移行済みなら再実行しない
- 旧データのバックアップを保持する

## 11. 完了条件

V3 初回リリースは、以下を満たしたときに完了と判断する。

- `setup -> home` の開始導線が安定して動く
- `EXP / きずな / 熟練` が表示だけでなく保存データにも反映される
- `ものがたり / そだてる` 分離が成立している
- リザルトで「今回何が進んだか」が分かる
- V2 セーブ移行で既存データを壊さない
- 手動確認、自動確認、ビルド確認が通る
