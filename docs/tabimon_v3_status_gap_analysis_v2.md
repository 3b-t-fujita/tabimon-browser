## 1. 結論サマリー

今回 AI エージェントが提示した `spec_v3_overview.md` と `spec_v3_technical.md` は、**初回 V3 のスコープ固定**という意味ではかなり予定に沿っています。  
特に、初回 V3 の対象を **EXP / きずな / 熟練 / ステージ分離 / ホーム改修 / 仲間詳細改修 / ステージ選択改修 / リザルト改修 / 軽量進化演出 / V2→V3 マイグレーション** に限定し、分岐進化・図鑑本格改修・設定画面・BGM/SE・PWA・`/evolution` 新規ページなどを対象外とした点は、事前の意思決定シートおよび実装指示書と整合しています。 fileciteturn7file9 fileciteturn7file11 fileciteturn7file3

ただし、**「実装済み」かどうかは今回の資料だけでは確証できない**状態です。  
理由は、`spec_v3_technical.md` は実装対象ファイル名やポリシー名、UseCase 名まで記載している一方で、**build 成功ログ、テスト結果、画面確認結果、差分一覧、PR/コミット、スクリーンショット** は提示されていないからです。したがって、今回の資料から言えるのは「実装仕様と実装予定はかなり整理された」ことであり、「本当にコードが完成した」とまでは断定できません。 fileciteturn7file14 fileciteturn7file17

さらに、今回ユーザー確認で判明した重要な差分として、**ホーム画面に V3 の中核である EXP / きずな のゲージ要約がまだ出ていない**点があります。  
これは軽微な見た目差分ではなく、`spec_v3_overview.md` に明記された「ホームで相棒の `EXP / きずな` 要約を表示する」という要件と不一致です。ホームは V3 の顔であり、ここにゲージがない状態は、V3 初回リリースの完了条件に対して未達とみなすべきです。 fileciteturn7file9 fileciteturn7file14

## 3. 予定と一致している点

### 3-1. 初回 V3 のスコープ固定

以下は、事前の意思決定どおりに揃っています。

- EXP 表示強化
- きずなシステム
- スキル熟練
- ステージ選択の `ものがたり / そだてる` 分離
- ホーム / 仲間詳細 / ステージ選択 / リザルト改修
- 軽量進化演出
- V2 → V3 マイグレーション

また、対象外として以下を外している点も一致しています。

- 分岐進化の本実装
- きずな進化
- スキル選択分岐
- 本格ストーリー多話化
- 図鑑本格改修
- 設定画面改修
- BGM / SE / PWA 強化
- `/evolution` 新規ページ
- 大量アニメーション / 大量アセット追加

これは、当初の「初回は軽量版で成立を優先する」という方針と整合しています。 fileciteturn7file9 fileciteturn7file11 fileciteturn7file3

また、`bondRank` は再計算可能、熟練の主データは `useCount`、という思想も揃っています。 fileciteturn7file14 fileciteturn7file17 fileciteturn7file4

このあたりは、予定側で固めた方針とほぼ一致しています。特に「既進化個体は維持」は重要な論点でしたが、今回の正本でも維持寄りで揃っています。 fileciteturn7file17 fileciteturn7file4

## 4. 予定との相違点・弱くなった点

### 4-1. 「実装完了の証拠」が不足している

`spec_v3_technical.md` には、実装ファイルやポリシー名、UseCase 名、画面コンポーネント名まで書かれています。  
しかし、以下が今回の提示物には含まれていません。

- build 成功結果
- unit test 実行結果
- Playwright 実行結果
- 手動確認結果
- 変更ファイル一覧
- 実際の差分内容
- 画面スクリーンショット
- PR / commit / patch の証跡

そのため、現状は **「実装仕様がかなり具体化された状態」** とは言えるが、**「完了確認済みの実装成果」** と断定するには材料が足りません。  
これは今回最大の注意点です。 fileciteturn7file14

### 4-2. ホーム画面のゲージ要約が未反映

ユーザー確認により、**ホーム画面に EXP やきずなのゲージが追加されていない**ことが判明しています。  
これは `spec_v3_overview.md` の以下の要件と明確に不一致です。

- 「ホームで相棒の `EXP / きずな` 要約を表示する」
- 「相棒を主役として表示」しつつ「数値面でも意味のある導線にする」

つまり、今のホームは「見た目上の主役表示」は入っていても、V3 で追加した成長軸の要約表示が不足している可能性が高いです。  
ホームは毎回最初に触る画面なので、ここにゲージがないと、V3 の価値である「毎回の成長実感」が薄れます。これは優先度の高い未実装、または未反映項目です。 fileciteturn7file9 fileciteturn7file14

### 4-3. きずなの具体加算仕様が一部後退している

事前の V3 統合仕様では、きずな加算はかなり具体的でした。  
例として、ホームタップ `+1`、相棒参加クリア `+8`、失敗 `+5`、ものがたり初回クリア `+15`、1 日初回起動 `+3`、進化完了 `+30` などが明示されていました。さらに実装前判断ログでも、**進化後 `きずな +30` は固定で実装**する推奨が置かれていました。 fileciteturn7file15 fileciteturn7file12

一方、今回の `spec_v3_overview.md` / `spec_v3_technical.md` では、冒険成功 / 失敗 / リタイア時は「定数加算」としか書かれておらず、**進化完了 `+30` や初回起動 `+3` の扱いが正本上は消えています**。  
このため、実装者が「最低限ホームタップ + 冒険結果だけ入れればよい」と解釈する余地があり、予定より機能が薄くなる可能性があります。 fileciteturn7file19 fileciteturn7file17

### 4-4. `そだてる` ステージの具体仕様が薄くなっている

以前の V3 統合仕様では、`そだてる` 側の例として以下がかなり具体的に出ていました。

- `けいけんちの草原 Easy / Normal / Hard`
- `しんか素材の洞窟`
- `なかまさがしの森`
- それぞれの `baseExp`
- 目安時間

しかし今回の正本では、`EXP / しんか素材 / なかまさがし` の意図が分かる育成用ステージを表示する、という説明に留まっています。  
`StageMasterV3` に `farmCategory` や `estimatedMinutes` はあるものの、**初回リリースにどの FARM ステージを何本入れるのか、マスタ値がどう確定しているのか** までは見えません。  
そのため、実装時にステージ本数や報酬値が暫定のまま進むリスクがあります。 fileciteturn7file10 fileciteturn7file19 fileciteturn7file14

### 4-5. リザルト画面の詳細粒度が簡略化されている

実装指示では、リザルトは最優先画面であり、EXP・きずな・熟練・ドロップ・進化通知・CTA を明確に出すことが求められていました。詳細 UI 仕様でも、セクション順、アニメーション、通知文言などがかなり詰められていました。 fileciteturn7file14

一方、今回の `spec_v3_overview.md` は「EXP セクション / きずなセクション / 熟練セクション / 進化通知または軽量進化演出 / CTA」というレベルの整理で、**ドロップセクションの明記が弱い**です。また、Before/After 演出やレベルアップ通知、きずな段階アップ通知の詳細ルールも overview には残っていません。  
技術仕様側で `ResultSummaryView.tsx` や戻り値定義は示されているが、**UI の完成基準が以前より薄い**ため、レビュワーが期待する絵と実装者の解釈がズレやすいです。 fileciteturn6file1 fileciteturn7file14

### 4-6. ホーム軽量演出の上限が正本ではあいまい

実装前判断ログでは、ホームのリアクションは「軽量版のみ。表情差分 + 吹き出し + 小エフェクトまで」と上限を切っていました。  
一方、今回の正本では「相棒タップで軽いリアクション兼 `きずな +1`」までしか書かれていません。  
これは悪いことではないですが、**演出粒度の上限が明文化されなくなったことで、AI エージェントが後で勝手に増やす / 逆に最小実装に寄せすぎる**余地が残っています。 fileciteturn7file3 fileciteturn7file11

### 4-7. URL / クエリ方針の細部が消えている

事前の整理では、`ものがたり / そだてる` のタブ状態は URL クエリに持たず、まずは UI state だけで扱う、という方向がありました。  
今回の正本では `/adventure/stages` を 2 タブにすることは示されているが、**URL に持つか持たないかの方針は明示されていません**。  
この論点は細かいですが、E2E や戻る操作、深リンク、今後の拡張に影響します。現時点で未固定のまま見えます。 fileciteturn7file4 fileciteturn7file19

## 5. ホーム画面ゲージ未反映への追加分析と実装提案

### 5-1. なぜ重要か

ホームは、プレイヤーが最初に見る V3 の代表画面です。ここで相棒の `EXP / きずな` が見えないと、以下の問題が起きます。

- V3 の追加価値が一目で伝わらない
- リザルトで増えた値がホームに戻っても確認できず、成長の連続性が切れる
- 仲間詳細に行かない限り、V3 の新軸を感じにくい
- 「相棒を主役にする」という UI と「相棒に数値的意味を持たせる」という設計が分離してしまう

つまり、ホームゲージ未反映は単なる UI 漏れではなく、**V3 体験の入口欠損**です。 fileciteturn7file9

### 5-2. 実装ゴール

ホームでは、詳細値を全部見せるのではなく、**要約としての見せ方**に徹するのが適切です。実装ゴールは次のとおりです。

- 相棒カード内に `EXP` の簡易ゲージを表示する
- 同じく `きずな` の簡易ゲージを表示する
- `EXP` は「現在Lv」「次Lvまでの進捗」が一目で分かる
- `きずな` は「現在ランク」「次ランクまでの進捗」が一目で分かる
- 詳細値・熟練・スキル別内訳は仲間詳細に逃がす
- 相棒カード全体押下で仲間詳細へ遷移する
- 相棒本体タップは軽いリアクション + `きずな +1` に使い、詳細遷移と競合しないようにする

### 5-3. 推奨UI構成

相棒エリアの下、主CTAの上に以下のような `成長サマリーカード` を 1 枚追加するのが分かりやすいです。

```text
┌─────────────────────────────┐
│  ヒノタネ  相棒                 │
│  Lv10                        ＞ │  ← カード全体で詳細へ
│  EXP  ███████░░░  420 / 600     │
│  きずな ★★☆☆   █████░░░░ 162/250 │
└─────────────────────────────┘
```

推奨表示ルール:

- 1行目: モンスター名 + 相棒チップ
- 2行目: `Lv` と詳細遷移の矢印
- 3行目: `EXP` ラベル + バー + 数値
- 4行目: `きずな` ラベル + ランク表示 + バー + 数値

### 5-4. ViewModel に持たせるべき項目

`buildHomeViewModelUseCase.ts` で、少なくとも以下を組み立てるのがよいです。

```ts
interface HomeGrowthSummaryViewModel {
  mainMonsterId: string;
  mainMonsterName: string;
  level: number;
  currentExp: number;
  nextLevelRequiredExp: number;
  expProgressRatio: number; // 0..1
  bondPoints: number;
  bondRank: 0 | 1 | 2 | 3 | 4;
  nextBondRequiredPoints: number | null; // rank4ならnull
  bondProgressRatio: number; // 0..1
  bondCurrentFloor: number;
  bondCurrentCeil: number | null;
}
```

補助関数として:

- `getExpProgress(level, currentExp)`
- `getBondRank(bondPoints)`
- `getBondProgress(bondPoints, bondRank)`

を `Domain/Application` 側に寄せ、UI 側では計算しない構成が安全です。これは V3 技術方針とも一致します。 fileciteturn7file14

### 5-5. UIコンポーネント実装案

`HomeScreenPatternStitch.tsx` に直接ロジックを散らさず、専用の小コンポーネントを 1 つ噛ませるのがおすすめです。

候補:

- `src/components/home/MainMonsterGrowthSummaryCard.tsx`

責務:

- 受け取った ViewModel を表示するだけ
- バーの見た目、ラベル、押下UIを担当
- ロジック計算は持たない

props 例:

```ts
interface MainMonsterGrowthSummaryCardProps {
  summary: HomeGrowthSummaryViewModel;
  onOpenDetail: () => void;
}
```

### 5-6. 具体的な実装手順

1. `buildHomeViewModelUseCase.ts` に `growthSummary` を追加する  
2. `bondPolicy.ts` または専用 helper で `bondProgressRatio` を返せるようにする  
3. レベル経験値テーブルから `nextLevelRequiredExp` と `expProgressRatio` を算出する helper を追加する  
4. `HomeScreenPatternStitch.tsx` で相棒カード直下に `MainMonsterGrowthSummaryCard` を挿入する  
5. カード押下で `/monsters/[id]` に遷移させる  
6. 相棒本体タップの `+1` 処理と押下領域が競合しないように、画像/立ち絵部分だけを別の押下ターゲットにする  
7. ホーム再描画時に、リザルト後の値が即座に反映されるか確認する  
8. Playwright にホームゲージ表示確認を追加する

### 5-7. 文言・見せ方の推奨

EXP:

- `EXP`
- `あと 180 で Lv11` を補助文で出してもよい

きずな:

- `きずな`
- `ランク2`
- `次まで 88`

ただしホームは要約画面なので、文言は増やしすぎず、**ラベル + バー + 最小限の数値**で留めるのがよいです。

### 5-8. 受け入れ条件

ホームゲージ実装の完了条件は以下です。

- 相棒がいる場合、ホームに `EXP` ゲージが表示される
- 相棒がいる場合、ホームに `きずな` ゲージが表示される
- リザルト後に戻ると増加後の値が反映される
- 相棒カードから仲間詳細へ遷移できる
- 相棒タップの `きずな +1` と詳細遷移が誤爆しない
- 相棒未設定や旧セーブでもクラッシュしない
- Playwright または手動確認で表示が確認済み

## 6. まだ実装できていない可能性が高い、または少なくとも未検証の部分

以下は、**未実装と断定はできないが、今回の資料だけでは完了確認できない項目**です。

### 6-1. build / test / E2E の実行結果

技術仕様書には、`npm test`、`npx playwright test`、`npm run build` を通すことがリリース条件として書かれています。  
ただし、今回それらの**実行結果は提出されていません**。  
したがって、少なくとも「完了済み確認」はまだできていません。 fileciteturn6file1

### 6-2. `そだてる` 側の実マスタ投入

`StageMasterV3` の型定義はあるが、**実際のステージマスタに FARM ステージが何本入り、どう解放され、どう報酬付与されるのか** は今回の資料では確認できません。  
型だけ入って、データ投入が未完、というパターンは十分あり得ます。 fileciteturn7file14 fileciteturn7file10

### 6-3. リザルトの「嬉しさ」の作り込み

リザルトの戻り値項目はかなり整理されているが、V3 の核は「今回何が進んだかが嬉しく分かること」でした。  
そこに対して、実際に UI がどの粒度まで作り込まれたかは、画面キャプチャや動作確認ログがないため分かりません。  
つまり、**機能反映は入っていても、体験品質までは未確認**です。 fileciteturn7file13 fileciteturn7file14

### 6-4. Manual verification の完了

技術仕様書には `v3_manual_verification_checklist.md` が関連資料として挙がっているが、今回はその内容や実施結果が提示されていません。  
そのため、開始導線、相棒表示、詳細画面、ステージ分離、リザルト、マイグレーションなどの**手動確認が実際に終わったかは不明**です。 fileciteturn6file1
