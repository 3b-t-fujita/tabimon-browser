# タビモン V3 リザルト画面 UI 仕様

> 更新日: 2026-03-27
> 対象画面: `/adventure/result`
> 前提: spec_v3.md（統合版）
> 位置づけ: V3 最重要改修画面の詳細 UI 仕様

---

## 1. この画面の役割

リザルト画面は V3 で**最も重要な改修対象**である。

V2 では「経験値と候補の反映」が役割だったが、V3 では**3 つの成長軸すべての進捗を 1 画面で可視化し、「この冒険で何が育ったか」を伝える画面**に変わる。

プレイヤーがこの画面で感じるべきことは:

> **「今回も何かが進んだ。次も冒険したい。」**

---

## 2. 画面遷移上の位置

```
バトル勝利 or 探索完了
  │
  ▼
リザルト画面（本画面）
  │
  ├─ 候補あり → /adventure/candidate
  └─ 候補なし → /home
```

失敗・リタイア時もリザルト画面を経由する（EXP は係数で減るが、きずなは別途加算）。

---

## 3. 画面構成の全体像

リザルト画面は**縦スクロール 1 画面**で、上から順に以下のセクションを表示する。

```
┌──────────────────────────────────┐
│  ① ヘッダー                      │
├──────────────────────────────────┤
│  ② けいけんちセクション          │
├──────────────────────────────────┤
│  ③ きずなセクション              │
├──────────────────────────────────┤
│  ④ スキルセクション              │
├──────────────────────────────────┤
│  ⑤ ドロップセクション            │
├──────────────────────────────────┤
│  ⑥ フッター（CTA）              │
└──────────────────────────────────┘
```

### スクロール方針

- 基本は 1 画面に収まることを目指す
- 情報量が多い場合（複数モンスター＋ドロップ多数）はスクロール可
- CTA ボタンは画面下部固定（スクロールに追従しない）

## 4. 各セクション詳細

### 4.1 ① ヘッダー

#### 成功時

```
┌──────────────────────────────────┐
│                                   │
│         冒険クリア！              │
│    [ステージ名: けいけんちの草原] │
│    [難易度アイコン: ★★☆ Normal] │
│                                   │
└──────────────────────────────────┘
```

#### 失敗時

```
┌──────────────────────────────────┐
│                                   │
│          おつかれさま！           │
│       つぎはきっとうまくいく      │
│    [ステージ名] [難易度]         │
│                                   │
└──────────────────────────────────┘
```

#### 仕様

| 項目 | 仕様 |
|---|---|
| タイトル（成功） | 「冒険クリア！」固定 |
| タイトル（失敗） | 「おつかれさま！」固定 |
| サブテキスト（失敗） | 「つぎはきっとうまくいく」固定 |
| ステージ名 | ステージマスターの `name` |
| 難易度 | Easy/Normal/Hard をアイコン＋テキストで表示 |
| 背景 | 成功時は明るい、失敗時はやや落ち着いたトーン |
| 初回クリア | 「はじめてクリア！」バッジを追加表示 |

---

### 4.2 ② けいけんちセクション

**最も面積を使うセクション。** パーティ全員の EXP 変化を表示する。

#### レイアウト

```
┌──────────────────────────────────┐
│  ── けいけんち ──                 │
│                                   │
│  [アイコン] ヒノタネ  相棒       │
│  +96 EXP (相棒ボーナス +20%)     │
│  ██████░░░ → ██████████░ (68→82%)│
│                                   │
│  [アイコン] カゼマル              │
│  +80 EXP                         │
│  ████░░░░░ → ██████░░░ (42→53%) │
│                                   │
│  ※ ヒノタネはあと126でレベルアップ │
│                                   │
└──────────────────────────────────┘
```

#### レベルアップ時

```
┌──────────────────────────────────┐
│  ── けいけんち ──                 │
│                                   │
│  [アイコン] ヒノタネ  相棒       │
│  +96 EXP (相棒ボーナス +20%)     │
│  ████████░░ → ██████████████ 100%│
│                                   │
│  ┌────────────────────────┐      │
│  │  🎉 レベルアップ！     │      │
│  │  Lv 10 → Lv 11        │      │
│  └────────────────────────┘      │
│                                   │
│  あたらしいゲージ:               │
│  ██░░░░░░░░░░░░ (26 / 750)  3%  │
│  つぎの節目: Lv 15 → ★しんか！  │
│                                   │
└──────────────────────────────────┘
```

#### 各要素の仕様

| 要素 | 仕様 |
|---|---|
| モンスターアイコン | 32×32px、仲間一覧と同じアイコン |
| モンスター名 | マスター名を表示 |
| 相棒ラベル | 相棒設定中のモンスターのみ「相棒」チップ表示 |
| 獲得 EXP | `+{数値} EXP` で表示。相棒ボーナス適用時は補足テキスト |
| EXP ゲージ | `SoftCard` 内にバーを配置 |
| バー色 | 通常: ブルー系、90% 以上: イエロー系 |
| レベルアップ通知 | `SoftCard` 内にインライン通知。背景色を変えて目立たせる |
| 次の節目 | レベルアップ後のみ表示。進化/スキル習得までの距離 |
| 表示順 | 相棒が最上位。以降はパーティ順 |
| 助っ人 | EXP 対象外のため表示しない |

#### バーアニメーション仕様

| パラメータ | 値 |
|---|---|
| アニメーション方式 | 左端から右へ、before → after に伸びる |
| 基本時間 | 0.6 秒 |
| イージング | ease-out |
| 大量 EXP（25% 以上増） | 0.8 秒に延長 + 「ぐんと成長した！」テキスト |
| 90% 到達 | バー色がイエローに遷移（0.3 秒フェード） + 「もうすぐレベルアップ！」 |
| 100% 到達（Lv アップ） | バー満タン → 0.3 秒ホールド → 光フラッシュ → バーリセット → 新ゲージ表示 |
| 複数 Lv アップ | 満タン→リセット→再充填を繰り返す（各 0.5 秒） |

#### テキスト演出ルール

| 条件 | 表示テキスト | タイミング |
|---|---|---|
| 通常 | なし | ― |
| 25% 以上増加 | 「ぐんと成長した！」 | バーアニメ完了時 |
| 90% 到達 | 「もうすぐレベルアップ！」 | バーが 90% に達した時 |
| Lv アップ | 「🎉 レベルアップ！ Lv X → Lv Y」 | 光フラッシュ後 |
| 初回クリア | 「はじめてクリア！ ボーナス +{値}」 | EXP 表示時 |

### 4.3 ③ きずなセクション

**相棒のみ表示。** 相棒以外のモンスターのきずな変化はこの画面では省略する。

#### 通常時

```
┌──────────────────────────────────┐
│  ── きずな ──                     │
│                                   │
│  ♥ ヒノタネ +8                   │
│  [☆☆░░] 162 / 250               │
│  きずな3まで あと88              │
│                                   │
└──────────────────────────────────┘
```

#### きずな段階アップ時

```
┌──────────────────────────────────┐
│  ── きずな ──                     │
│                                   │
│  ♥ ヒノタネ +8                   │
│  ┌────────────────────────┐      │
│  │  ♥♥ きずなが ☆☆ に！  │      │
│  │  冒険のはじまりに       │      │
│  │  おうえんしてくれるよ   │      │
│  └────────────────────────┘      │
│  [☆☆░░] 154 / 400               │
│                                   │
└──────────────────────────────────┘
```

#### 失敗時

```
┌──────────────────────────────────┐
│  ── きずな ──                     │
│                                   │
│  ♥ ヒノタネ +5                   │
│  いっしょにがんばったきずな       │
│  [☆░░░] 38 / 50                 │
│                                   │
└──────────────────────────────────┘
```

#### 仕様

| 要素 | 仕様 |
|---|---|
| 表示対象 | 相棒のみ |
| ポイント増加表示 | `♥ {名前} +{値}` |
| きずなバー | 4 段階の区切りマーク付きバー |
| 段階アイコン | ☆ で段階数を表示（MAX は ★） |
| 残量テキスト | `きずな{次段階}まで あと{値}` |
| 段階アップ通知 | `SoftCard` 内インライン。段階ごとの解放内容を 1 行で説明 |
| 失敗時 | +5 表示 + 「いっしょにがんばったきずな」テキスト |
| アニメーション | バーが伸びる（0.4 秒、ease-out） |

#### きずなバーの区切り表示

```
[====|========|================|..........................]
 0   50      150              400                       1000
 ☆    ☆☆     ☆☆☆             ★MAX
```

小さなドットやラインで段階境界を示す。現在どの区間にいるかが直感的に分かる。

---

### 4.4 ④ スキルセクション

**段階が変わったスキルだけ表示する。** 変化がなければこのセクション自体を非表示にする。

#### 熟練アップ時

```
┌──────────────────────────────────┐
│  ── スキル ──                     │
│                                   │
│  ⚔ ほのおパンチが ★★ になった！ │
│  つかうほどに コストがへるよ      │
│                                   │
└──────────────────────────────────┘
```

#### 変化なし（通常の進行表示）

```
┌──────────────────────────────────┐
│  ── スキル ──                     │
│                                   │
│  ⚔ ほのおパンチ 32/40回         │
│    ★★ まであと8回               │
│                                   │
└──────────────────────────────────┘
```

#### 仕様

| 要素 | 仕様 |
|---|---|
| 表示条件（段階アップ） | 熟練段階が変わったスキルがある場合 |
| 表示条件（通常進行） | 段階アップがない場合、相棒の主力スキル 1 つの進行を表示 |
| 段階アップ通知テキスト | `{スキル名}が {段階表示} になった！` |
| 段階ごとの説明 | ★:「エフェクトがかわったよ」★★:「コストがへるよ」★★★:「いりょくアップ！」 |
| 通常進行テキスト | `{スキル名} {現在回数}/{次段階回数}回  {段階}まであと{残り}回` |
| 表示スキル数 | 最大 2 つ（段階アップは全て表示、通常進行は 1 つ） |
| セクション非表示 | 段階アップも通常進行も表示不要な場合（相棒がいない等） |

---

### 4.5 ⑤ ドロップセクション

素材や候補モンスターが出た場合に表示する。何もなければ非表示。

#### レイアウト

```
┌──────────────────────────────────┐
│  ── もちもの ──                   │
│                                   │
│  🔥 ほのおの石 ×1                │
│  🌿 みどりのしずく ×2            │
│                                   │
└──────────────────────────────────┘
```

#### 仕様

| 要素 | 仕様 |
|---|---|
| 表示条件 | ドロップアイテムがある場合のみ |
| アイテム表示 | アイコン + 名前 + 個数 |
| 候補モンスター | 「あたらしい なかまの候補が！」として別枠表示 → CTA で `/adventure/candidate` へ |
| 表示順 | 進化素材 → その他アイテム → 候補通知 |

---

### 4.6 ⑥ フッター（CTA）

画面下部固定。スクロールに追従しない。

#### 候補なし時

```
┌──────────────────────────────────┐
│                                   │
│         [ つぎへ ]                │
│                                   │
└──────────────────────────────────┘
```

#### 候補あり時

```
┌──────────────────────────────────┐
│                                   │
│      [ なかまを見にいく ]         │
│                                   │
└──────────────────────────────────┘
```

#### 仕様

| 要素 | 仕様 |
|---|---|
| ボタンスタイル | `PrimaryButton`（V2 共通部品） |
| 候補なし | 「つぎへ」→ `/home` へ遷移 |
| 候補あり | 「なかまを見にいく」→ `/adventure/candidate` へ遷移 |
| 位置 | 画面下部固定（`position: sticky; bottom: 0`） |
| 背景 | 白のグラデーションフェードで本文と分離 |

---

## 5. 表示順序とアニメーションのタイムライン

リザルト画面は上から順に**段階的に表示**する。全てを一度に出さない。

### 5.1 タイムライン

| 経過時間 | 表示内容 | アニメーション |
|---|---|---|
| 0.0 秒 | ① ヘッダー表示 | フェードイン（0.3 秒） |
| 0.3 秒 | ② けいけんちセクション表示開始 | スライドアップ + フェードイン |
| 0.5 秒 | ② 相棒の EXP バーアニメ開始 | バー伸長（0.6〜0.8 秒） |
| 1.1 秒 | ② 相棒の演出テキスト表示 | フェードイン |
| 1.3 秒 | ② 2 体目の EXP バーアニメ | バー伸長 |
| 1.8 秒 | ② Lv アップがあれば通知表示 | ポップイン（scale 0.8→1.0） |
| 2.2 秒 | ③ きずなセクション表示 | スライドアップ + バー伸長 |
| 2.8 秒 | ③ きずな段階アップがあれば通知 | ポップイン |
| 3.0 秒 | ④ スキルセクション表示（あれば） | スライドアップ |
| 3.3 秒 | ⑤ ドロップセクション表示（あれば） | スライドアップ |
| 3.5 秒 | ⑥ CTA ボタン表示 | フェードイン |

### 5.2 タップでスキップ

| 条件 | 挙動 |
|---|---|
| アニメ途中にタップ | 全アニメを即完了し、最終状態を表示 |
| 全アニメ完了後にタップ | CTA と同じ遷移を実行 |

### 5.3 合計所要時間

- 通常（段階アップなし）: 約 2.5 秒
- フル（Lv アップ＋きずな段階＋スキル熟練＋ドロップ）: 約 3.5 秒
- スキップ時: 即座

### 6.2 最も頻繁に見るパターン

- **パターン A（通常クリア）**: 最も多い。ゲージが伸びるだけだが、毎回確実に進む。
- **パターン D（スキル熟練アップ）**: 2〜7 周に 1 回。短いスパンで達成感。
- **パターン B（Lv アップ）**: 3〜21 周に 1 回。最大の報酬感。

### 6.3 最も印象に残るパターン

- **パターン G（全部盛り）**: 初回クリア＋ Lv アップ＋きずな段階＋熟練が同時に来る。めったにないが、来た時の満足感が非常に高い。

## 7. ViewModel

### 7.1 ResultViewModel

```ts
interface ResultViewModel {
  // ヘッダー
  stageName: string;
  difficulty: 'EASY' | 'NORMAL' | 'HARD';
  result: 'SUCCESS' | 'FAILURE' | 'RETIRE';
  isFirstClear: boolean;

  // けいけんち
  expEntries: ExpEntry[];

  // きずな（相棒のみ）
  bond: BondEntry | null;

  // スキル熟練
  skillChanges: SkillChangeEntry[];
  skillProgress: SkillProgressEntry | null;  // 段階変化なし時の通常進行

  // ドロップ
  drops: DropEntry[];
  hasCandidate: boolean;
}

interface ExpEntry {
  monsterId: string;
  monsterName: string;
  monsterIconUrl: string;
  isPartner: boolean;
  gainedExp: number;
  partnerBonusApplied: boolean;
  beforeExp: number;      // アニメ前の蓄積 EXP
  afterExp: number;       // アニメ後の蓄積 EXP
  requiredExp: number;    // 次 Lv の必要 EXP
  beforePercent: number;  // アニメ前の %
  afterPercent: number;   // アニメ後の %
  leveledUp: boolean;
  beforeLevel: number;
  afterLevel: number;
  newGaugePercent: number | null;  // Lv アップ後の新ゲージ %
  nextMilestone: string | null;    // 「Lv15 → ★しんか！」等
  textEffect: 'NONE' | 'BIG_GROWTH' | 'ALMOST_LEVELUP' | 'FIRST_CLEAR';
}

interface BondEntry {
  monsterName: string;
  gained: number;         // +8 or +5（失敗時）
  beforePoints: number;
  afterPoints: number;
  currentRank: number;
  nextRankRequired: number;
  rankedUp: boolean;
  newRank: number | null;
  newRankDescription: string | null;  // 「冒険のはじまりに おうえんしてくれるよ」
  isFailureGain: boolean; // 失敗時の獲得か
}

interface SkillChangeEntry {
  skillName: string;
  newStage: 1 | 2 | 3;
  stageDescription: string;  // 「コストがへるよ」等
}

interface SkillProgressEntry {
  skillName: string;
  currentUses: number;
  nextStageUses: number;
  remaining: number;
  nextStageLabel: string;   // 「★★」等
}

interface DropEntry {
  type: 'ITEM' | 'CANDIDATE_NOTICE';
  iconUrl: string;
  name: string;
  count: number;
}
```

### 7.2 ViewModel 生成元

| フィールド | 生成元 |
|---|---|
| expEntries | `adventureSession` の結果 + `BALANCE_V3.EXP_TABLE` + 補正計算 |
| bond | `OwnedMonsterV3.bondPoints` の差分 + `BALANCE_V3.BOND.STAGES` |
| skillChanges | バトル中のスキル使用カウント + `BALANCE_V3.SKILL_PROFICIENCY.STAGES` |
| drops | `adventureSession` のドロップ結果 |

## 8. コンポーネント構成

```
ResultScreen
├── ResultHeader
├── ExpSection
│   └── ExpMonsterRow (× パーティ人数、相棒が最上位)
│       ├── MonsterIcon
│       ├── ExpGainLabel
│       ├── ExpBar (アニメーション付き)
│       ├── LevelUpNotice (条件付き表示)
│       └── NextMilestoneLabel (条件付き表示)
├── BondSection (相棒のみ、条件付き表示)
│   ├── BondGainLabel
│   ├── BondBar
│   ├── BondRankUpNotice (条件付き表示)
│   └── BondRemainingLabel
├── SkillSection (条件付き表示)
│   ├── SkillRankUpNotice (段階アップ時)
│   └── SkillProgressLabel (通常進行時)
├── DropSection (条件付き表示)
│   └── DropItemRow (× ドロップ数)
└── ResultFooter (固定)
    └── PrimaryButton
```

### 既存共通部品の利用

| V2 共通部品 | 使用箇所 |
|---|---|
| `AppScreenHeader` | 使用しない（リザルトは独自ヘッダー） |
| `SoftCard` | 各セクションの囲み |
| `UiChip` | 「相棒」ラベル、難易度ラベル、初回クリアバッジ |

## 10. アクセシビリティとエッジケース

| ケース | 対応 |
|---|---|

## 11. 実装上のメモ

### 11.1 アニメーション実装

- `ExpBar` のアニメは CSS `transition` + `width` プロパティで実装
- タイムライン制御は `setTimeout` チェーンまたは `requestAnimationFrame`
- スキップは state フラグで全アニメを即完了させる

### 11.2 Store との連携

- リザルト表示に必要なデータは `resultStore` に格納
- バトル完了時に `BuildResultViewModelUseCase` が ViewModel を生成
- きずな・熟練度の永続化は CTA タップ後（`CommitResultUseCase`）で実行
- 永続化前にリザルト画面を離脱した場合のリカバリは `pendingResult` で管理

### 11.3 テスト観点

- [ ] 通常クリア時にゲージが正しく伸びる
- [ ] Lv アップ時にバー満タン→リセット→新ゲージが表示される
- [ ] 複数 Lv アップが正しく処理される
- [ ] きずな段階アップ通知が正しいタイミングで出る
- [ ] 失敗時にきずな +5 が表示される
- [ ] スキル熟練アップ通知が出る
- [ ] 変化なし時にスキルセクションが非表示になる
- [ ] 候補ありの場合 CTA が「なかまを見にいく」に変わる
- [ ] タップスキップが全パターンで動作する
- [ ] 初回クリアボーナスが正しく表示される
