# Tabimon V3 リリース前 最終チェックリスト

> 更新日: 2026-03-30

## 目的

V3 初回リリースを GitHub へ反映する前に、実装・表示・確認資料がそろっているかを最終確認するためのチェックリストです。

## 1. コードとデータ

- `OwnedMonsterV3` の追加項目が保存・読込・移行で壊れていない
- `StageMasterV3` の `STORY / FARM`、`farmCategory`、`difficultyTier` が反映されている
- `そだてる` が `EXP / BOND / SKILL` の 6 ステージ構成になっている
- `そだてる` の候補モンスターが
  - 対応エリアのみ
  - 初期キャラ除外
  - レア除外
  - 等確率
 になっている
- `そだてる` の敵強さ補正が
  - 前半 `1.2倍`
  - 後半 `1.5倍`
 になっている

## 2. 画面表示

- `Stage` が `ステージ` に統一されている
- `Easy / Normal / Hard` が `かんたん / ふつう / むずかしい` に統一されている
- `たすっ人 / 助っ人` の表示が `おたすけ` に統一されている
- 子ども向けの短い文言にそろっている
- ホームに成長サマリーが出ている
- 相棒立ち絵タップと詳細遷移が競合していない

## 3. 主要導線

- `/setup` から開始できる
- `/home` が読み込み中のまま止まらない
- `/adventure/stages` の `ものがたり / そだてる` が切り替わる
- `/adventure/confirm` に編成とステージ名が正しく出る
- `/adventure/play` でイベントと分岐が進む
- `/adventure/battle` で相棒が主役に見える
- `/adventure/result` からホームへ戻ったときに成長反映が見える

## 4. 戦闘とイベント

- 被ダメージ時に対象だけが小さく揺れる
- 敵味方が完全同時に揺れすぎない
- ノード5のイベントでは戦闘イベントが発生しない
- ステージ終盤で戦闘が不自然に連続しない

## 5. V3 固有機能

- ホームの `EXP / きずな` 要約が見える
- ホームタップの `きずな +1` が 1日3回まで機能する
- 仲間詳細で `EXP / きずな / じゅくれん` が見える
- リザルトで `EXP / きずな / じゅくれん / 進化通知` が見える
- V2 → V3 移行データでも安全に表示される

## 6. 自動確認

- `npm test` 通過
- `npm run build` 通過
- `npx playwright test --reporter=list` 通過

## 7. GitHub 反映前の整理

- 生成物を含めない
  - `.playwright-cli/`
  - `playwright-report/`
  - `test-results/`
  - `.codex-next-start-*.log`
- ローカル専用設定や一時ログを混ぜない
- 仕様書と実装が大きくずれていない
- README から V3 正本にたどれる
