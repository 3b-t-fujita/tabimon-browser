# Tabimon プレイテストチェックリスト

> 更新日: 2026-03-30

## 目的

主要導線を 1 周プレイして、表示崩れ・導線崩れ・文言違和感を早めに見つけるための確認手順です。

## 事前確認

1. `npm test`
2. `npm run build`
3. 本番相当で確認する場合は `npm run start -- -H 127.0.0.1 -p 4303`
4. 開発モードで確認する場合は `npm run dev`

## 主要フロー

1. `/setup`
確認すること:
- 初期設定で名前とワールドを選べる
- `ぼうけんを はじめる` が押せる
- 完了後に `/home` へ進む

2. `/home`
確認すること:
- 相棒画像、仲間数、おたすけ数が表示される
- `ぼうけんへ いく`, `なかま`, `へんせい`, `こうかん` が押せる
- 成長サマリーに `EXP / きずな` が出る

3. `/party`
確認すること:
- 相棒と助っ人候補の表示が崩れない
- 助っ人追加・削除ができる
- 空状態の文言が自然

4. `/adventure/stages`
確認すること:
- `ものがたり / そだてる` を切り替えられる
- ワールドごとのカードが見やすい
- `ステージ` 表記が日本語でそろっている
- ステージ選択後に `/adventure/confirm` へ進む

5. `/adventure/confirm`
確認すること:
- 相棒と助っ人の編成が正しく反映される
- ステージ名が `ミドリの森 ステージ 1` のように表示される
- 難易度が `かんたん / ふつう / むずかしい` で出る
- `編成を変更` で `/party` に戻れる
- `この編成で出発する` で `/adventure/play` に進む

6. `/adventure/play`
確認すること:
- ノード、イベント、分岐、戦闘導線が正しく進む
- 文言が状況に合っている
- ノード5のイベントで、戦闘が連続発生しない

7. `/adventure/battle`
確認すること:
- 1 画面に敵と味方が収まる
- 相棒が中央手前で主役に見える
- スキルボタンの状態が分かる
- ダメージを受けた対象だけが小さく揺れる

8. `/adventure/result`
確認すること:
- 結果、経験値、レベルアップ、解放が見やすい
- `EXP / きずな / じゅくれん` が結果に応じて更新される
- 候補ありなら `/adventure/candidate` へ進む
- 候補なしなら CTA で `/home` に戻れる

9. `/adventure/candidate`
確認すること:
- 候補カードが見やすい
- `仲間にする` と `今回は見送る` の意味が明確

10. `/adventure/candidate/replace`
確認すること:
- 上限時の説明が分かる
- 相棒が手放し対象に出ない
- 入替確認ダイアログが自然

11. `/qr`, `/qr/generate`, `/qr/scan`, `/qr/confirm`
確認すること:
- 生成/読取の導線が迷わない
- 読取パネルの説明が自然
- 内容確認のチップ表示が崩れない
- `おたすけ` 表記がそろっている

## チェック観点

- 余白が狭すぎないか
- ローディング文言が状況に合っているか
- エラー時の戻り先が自然か
- CTA の優先順位が視覚的に明確か
- スマホ幅でスクロールが不必要に長くなっていないか
- `Stage / Easy / 助っ人 / ばしょ` の旧表記が残っていないか

## 備考

- 静的確認だけなら `/preview/ui` を使う
- 本番相当の安定確認は `http://127.0.0.1:4303` を使う
- 実際の状態遷移確認は通常導線で行う
