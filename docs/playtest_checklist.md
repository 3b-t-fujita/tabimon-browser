# Tabimon プレイテストチェックリスト

> 更新日: 2026-03-27

## 目的

主要導線を 1 周プレイして、表示崩れ・導線崩れ・文言違和感を早めに見つけるための確認手順です。

## 事前確認

1. `npm test`
2. `npm run build`
3. `npm run dev` または `npx next dev -H 0.0.0.0 -p 4100`

## 主要フロー

1. `/setup`
確認すること:
- 初期設定で名前、ワールド、初期相棒を選べる
- CTA が無効/有効で適切に切り替わる
- 完了後に `/home` へ進む

2. `/home`
確認すること:
- 相棒画像、仲間数、助っ人数が表示される
- `冒険へ出発`, `仲間一覧`, `編成`, `QR交換` が押せる

3. `/party`
確認すること:
- 相棒と助っ人候補の表示が崩れない
- 助っ人追加・削除ができる
- 空状態の文言が自然

4. `/adventure/stages`
確認すること:
- ワールドごとのカードが見やすい
- 解放済みと未解放の差が分かる
- ステージ選択後に `/adventure/confirm` へ進む

5. `/adventure/confirm`
確認すること:
- 相棒と助っ人の編成が正しく反映される
- `編成を変更` で `/party` に戻れる
- `この編成で出発する` で `/adventure/play` に進む

6. `/adventure/play`
確認すること:
- ノード、イベント、分岐、戦闘導線が正しく進む
- 文言が状況に合っている

7. `/adventure/battle`
確認すること:
- 1 画面に敵と味方が収まる
- 相棒が中央手前で主役に見える
- スキルボタンの状態が分かる

8. `/adventure/result`
確認すること:
- 結果、経験値、レベルアップ、解放が見やすい
- 候補ありなら `/adventure/candidate` へ進む
- 候補なしなら少し待って `/home` に戻る

9. `/adventure/candidate`
確認すること:
- 候補カードが見やすい
- `仲間にする` と `今回は見送る` の意味が明確

10. `/adventure/candidate/replace`
確認すること:
- 上限時の説明が分かる
- 相棒が手放し対象に出ない
- 入替確認ダイアログが自然

11. `/qr`, `/qr/generate`, `/qr/scan`, `/qr/confirm`
確認すること:
- 生成/読取の導線が迷わない
- 読取パネルの説明が自然
- 内容確認のチップ表示が崩れない

## チェック観点

- 余白が狭すぎないか
- ローディング文言が状況に合っているか
- エラー時の戻り先が自然か
- CTA の優先順位が視覚的に明確か
- スマホ幅でスクロールが不必要に長くなっていないか

## 備考

- 静的確認だけなら `/preview/ui` を使う
- 実際の状態遷移確認は通常導線で行う
