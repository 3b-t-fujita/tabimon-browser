# Playwright Test Plan

## Goal
- ブラウザ上で主要導線が最低限プレイ可能かを自動確認する
- `setup` のような「見た目は出るが操作できない」不具合を早期に検出する
- V3 の追加要素が主要画面で壊れていないことをスモーク確認する

## Initial Scope
1. 初回導線
   - `/setup` が開く
   - ワールド選択ができる
   - `冒険を始める` が動作する
2. ホーム導線
   - `/home` または初期設定後のホーム表示が成立する
   - 主要 CTA が表示される
3. ステージ選択
   - `/adventure/stages` が開く
   - `ものがたり / そだてる` タブが切り替わる
4. 静的プレビュー
   - `/preview/ui` で主要画面見本が表示される
5. 仲間詳細
   - `/monsters` から仲間詳細へ遷移できる
   - `EXP / きずな / じゅくれん` の3軸が見える
6. リザルト
   - `/preview/ui` のリザルト見本で主要セクションが表示される
   - `けいけんち / きずな / じゅくれん / 進化` が見える

## Design Principles
- まずはスモーク中心にして、壊れているかどうかを短時間で分かる形にする
- IndexedDB や既存セーブに依存しすぎない
- 失敗時に原因が追いやすいよう、URL と見出しと CTA を明示的に検証する
- 不安定な画面は `preview/ui` を併用してデザイン崩れと導線不具合を切り分ける

## Execution Policy
- Playwright は Chromium のスマホ相当幅で実行する
- 実行サーバーは `next build` + `next start` を優先し、`dev` 固有不具合と切り分ける
- 失敗時は修正担当が再現と修正を行い、再実行して収束させる
- 実画面で不安定な導線は、まず `preview/ui` で見本を確認してから本線へ戻す
