# タビモン V2 仕様書

> バージョン: V2（UI 一新版）
> 更新日: 2026-03-27
> 対象: `tabimon-browser`

## 1. この文書の位置づけ

本書は、タビモンの現行版である **V2（UI 一新版）** の企画・画面・体験仕様をまとめたものです。  
技術的な詳細は [spec_v2_technical.md](./spec_v2_technical.md) を参照してください。

旧仕様書である `spec_concept.md` と `spec_technical.md` は履歴資料として保持し、本書を現行の正本として扱います。

## 2. V2 の定義

V2 は、既存のプレイ構造を維持したまま、UI と画面体験を全面的に刷新したバージョンです。

- スマホ 1 画面前提の情報整理
- 子ども向けに寄せた日本語表記
- 相棒を中心にしたホームとバトルの見せ方
- 主要導線全体のデザイン統一
- 共通 UI 部品による保守性向上

## 3. V2 の目的

- 初めて触る子どもでも迷いにくい
- 相棒モンスターへの愛着が自然に高まる
- 1 画面ごとの役割が明快で、次の行動を選びやすい
- 冒険、育成、つながり体験が一つの世界観でつながって見える

## 4. 対象ユーザー

- 小学生〜中学生を中心とした、モンスター育成ゲームが好きなユーザー
- 保護者やライト層を含む、難しすぎない RPG を求めるユーザー
- 友だち同士で「見せる・受け取る」遊びを楽しみたいユーザー

## 5. コア体験

### 5.1 基本ループ

1. 相棒とホームで次の行動を選ぶ
2. 冒険先を決めて旅に出る
3. 探索、イベント、バトルを進める
4. リザルトで経験値や候補を受け取る
5. 編成や仲間管理を整える
6. 友だちとコード交換をして助っ人を増やす

### 5.2 V2 が特に強めた体験

- ホームで「相棒が主役」であることが一目で分かる
- バトルで「背景の中にモンスターが立つ」臨場感がある
- 画面ごとに CTA が明確で、迷ったときに選ぶべき行動が分かる

## 6. 画面構成

### 6.1 主要画面

| 画面 | URL | 役割 |
|---|---|---|
| 起動 | `/` | セーブ確認と振り分け |
| 初期設定 | `/setup` | 名前、ワールド、最初の相棒の設定 |
| ホーム | `/home` | 冒険、仲間、編成、コード交換の入口 |
| 仲間一覧 | `/monsters` | 仲間の確認 |
| 仲間詳細 | `/monsters/[id]` | ステータス確認、相棒設定、手放し |
| 編成 | `/party` | 助っ人編成 |
| ステージ選択 | `/adventure/stages` | 冒険先の選択 |
| 冒険確認 | `/adventure/confirm` | 出発前の最終確認 |
| 探索 | `/adventure/play` | ノード移動、分岐、イベント |
| バトル | `/adventure/battle` | 戦闘 |
| リザルト | `/adventure/result` | 経験値、解放、候補の反映 |
| 候補受取 | `/adventure/candidate` | 新しい仲間候補の選択 |
| 候補入替 | `/adventure/candidate/replace` | 上限時の入替受取 |
| コード交換 | `/qr` | 交換入口 |
| コード作成 | `/qr/generate` | 仲間のコード作成 |
| コード読取 | `/qr/scan` | 画像から読取 |
| 受取確認 | `/qr/confirm` | 受取内容の確認 |
| 静的プレビュー | `/preview/ui` | セーブ不要の確認用 |

### 6.2 主要導線

#### 冒険導線

`/setup` → `/home` → `/party` → `/adventure/stages` → `/adventure/confirm` → `/adventure/play` → `/adventure/battle` → `/adventure/result` → `/adventure/candidate`

#### つながり導線

`/home` → `/qr` → `/qr/generate`  
`/home` → `/qr` → `/qr/scan` → `/qr/confirm`

## 7. UI / UX 方針

### 7.1 デザイン原則

- 白ベース
- 淡い補助色
- 大きな角丸
- 強すぎない影
- 子ども向けに読みやすい日本語中心の表記
- 主操作は丸い CTA で明確化

### 7.2 ホーム画面

- 相棒画像を画面上部の主役として表示する
- 相棒情報枠には「相棒」「名前」「レベル」を表示する
- `冒険へ出発` を最優先 CTA とする
- `仲間一覧` `編成` `コード交換` は二次導線として整理する

### 7.3 バトル画面

- 1 画面に敵・味方・操作帯を収める
- 奥に敵、手前に味方を置き、中央の相棒を最も大きく見せる
- ワールド背景を見せつつ、情報は半透明パネルで読みやすくする

### 7.4 文字表記

- 子ども向けを優先し、見える英字 UI は極力使わない
- ラベルは短く、行動の意味が分かる表現を使う
- 1 行に収まりそうな見出しは文字サイズ調整で優先的に 1 行化する

## 8. ゲームルール上の整理

### 8.1 相棒

- プレイヤーには相棒が 1 体いる
- 相棒はホームの主役であり、冒険の中心である
- 相棒設定中の仲間は手放せない

### 8.2 仲間

- 育成対象
- 詳細確認、相棒設定、手放しが可能

### 8.3 助っ人

- 友だちとの交換で受け取る補助戦力
- 編成には使えるが、育成対象ではない
- 冒険時に最大 2 体まで編成する

### 8.4 コード交換

- 送る側は仲間からコードを作る
- 受け取る側は画像からコードを読む
- 受取前に内容確認を行う

## 9. V2 の非対象

本バージョンでは次を主目的にしない。

- 大規模なゲームルール変更
- ワールドやステージ数の拡張
- ネットワーク対戦
- サーバー同期型のアカウント管理

## 10. 運用ルール

- 旧 `PatternA / PatternB` は復帰用バックアップとして残す
- 正式導線は V2 の新 UI を使う
- 新規画面追加時は [ui_design_system.md](./ui_design_system.md) を優先参照する
- プレイ確認は [playtest_checklist.md](./playtest_checklist.md) に従う
