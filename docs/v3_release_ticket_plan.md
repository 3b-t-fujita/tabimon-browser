# Tabimon V3 リリース達成用ステップ・チケット一覧

## 目的
- V3 初回リリース要件を、未実装や不安定要素を残さず達成する
- 実装、検証、リリース判断をチケット単位で進められるようにする

## 現在地
- V3 の主要追加要素は実装済み
  - EXP 表示強化
  - きずなシステム
  - スキル熟練システム
  - `ものがたり / そだてる` 分離
  - ホーム / 仲間詳細 / ステージ選択 / リザルト改修
  - 軽量進化演出
  - V2 → V3 マイグレーション
- 自動確認
  - Vitest: 通過
  - Playwright: 主要スモーク通過
- ただし、既存セーブ読込の揺れや V3 本筋シナリオの確認密度はまだ十分ではない

## 今後必要なステップ

### Step 1. 主要導線の安定化を完了する
- `setup`
- `home`
- `adventure/stages`
- `result`

### Step 2. V3 データ反映の通し確認を完了する
- EXP
- きずな
- 熟練
- 初回クリア EXP
- 進化通知

### Step 3. 既存セーブ移行の安全性を固める
- V2 → V3 補完
- 再実行安全性
- バックアップ保持

### Step 4. E2E 確認範囲をリリース基準まで広げる
- 現在のスモークから、V3 の価値に直結するフローへ拡張する

### Step 5. 手動確認と自動確認の基準を揃える
- 手動確認シートと Playwright の観点を整合させる

### Step 6. リリース判断に必要な資料を閉じる
- 既知の制約
- 確認手順
- URL
- リスク

## チケット一覧

## V3-01 `setup/home` 安定化完了
- 目的:
  - 初回導線で止まらない状態にする
- 背景:
  - `dev/start` の差やクライアント初期化差で揺れが出ていた
- 作業内容:
  - `setup -> home` を `dev/start` 両方で確認
  - `home` 既存セーブ読込の停止条件を洗い出す
  - 必要なら読込フォールバックを整理する
- 完了条件:
  - 新規開始が安定して通る
  - 既存セーブで `/home` が読み込み中のまま止まりにくい
- 優先度:
  - Highest

## V3-02 V3 リザルト反映の通し確認
- 目的:
  - V3 の価値である「何が進んだか分かる」を本当に成立させる
- 作業内容:
  - 冒険完了後に EXP, きずな, 熟練, 初回クリアEXP, 進化通知を確認
  - 表示と実データが一致するか確認
  - 必要ならユースケーステストを追加する
- 完了条件:
  - リザルト表示と保存データの差異がない
- 優先度:
  - Highest

## V3-03 きずなシステム最終確認
- 目的:
  - ホームタップと冒険完了時のきずな加算が仕様どおり動くようにする
- 作業内容:
  - 1日3回上限の確認
  - 冒険成功 / 失敗 / リタイア時の加算確認
  - ランク更新確認
- 完了条件:
  - 加算値とランク更新が仕様どおり
- 優先度:
  - High

## V3-04 熟練システム最終確認
- 目的:
  - 相棒スキル使用回数が正しく記録・反映されるようにする
- 作業内容:
  - 戦闘中の使用回数記録確認
  - リザルト反映確認
  - 仲間詳細表示確認
- 完了条件:
  - 使用回数と段階が一貫して反映される
- 優先度:
  - High

## V3-05 ステージ選択 V3 仕様確認
- 目的:
  - `ものがたり / そだてる` と 6 ステージ FARM の意味が UI 上で明確に分かるようにする
- 作業内容:
  - タブ切替確認
  - `FARM` 6 ステージ表示確認
  - `前半 / 後半` 表示確認
  - 解放 / 未解放表示確認
- 完了条件:
  - タブごとの意図と FARM の育成軸が迷わず伝わる
- 優先度:
  - High

## V3-06 モンスター詳細 V3 表示確認
- 目的:
  - `EXP / きずな / 熟練` の3軸が確認しやすい状態にする
- 作業内容:
  - 主要モンスターで詳細表示確認
  - 相棒設定・手放し既存責務の維持確認
- 完了条件:
  - V3 情報と既存導線が共存している
- 優先度:
  - High

## V3-07 V2→V3 マイグレーション確認
- 目的:
  - 既存ユーザーのセーブを壊さない
- 作業内容:
  - V2 ライクなセーブパターンを複数作る
  - V3 項目補完確認
  - バックアップ保持確認
  - 再読込時の再移行防止確認
- 完了条件:
  - 移行で進行や所持データを壊さない
- 優先度:
  - Highest

## V3-08 Playwright 拡張
- 目的:
  - V3 リリース判断に必要なブラウザ確認を自動化する
- 作業内容:
  - `result`
  - `monster detail`
  - `home -> stages`
  - 必要なら migration 前後の起動確認
- 完了条件:
  - 主要 V3 導線が E2E で確認できる
- 優先度:
  - High

## V3-09 手動確認基準の整備
- 目的:
  - 主導確認時に見るべき点を漏れなくする
- 作業内容:
  - 手動確認チェックリストを最新化
  - URL, 手順, 成功条件を明文化
- 完了条件:
  - 非開発者でも同じ観点で確認できる
- 優先度:
  - Medium

## V3-10 リリースノート・既知事項整理
- 目的:
  - リリース判断をしやすくする
- 作業内容:
  - 既知の制約整理
  - 確認結果整理
  - README / docs 更新
- 完了条件:
  - リリース時に参照すべき情報が1箇所にまとまる
- 優先度:
  - Medium
