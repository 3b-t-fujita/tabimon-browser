# タビモン V3 初回リリース確定版 意思決定シート

作成日: 2026-03-28  
対象: `tabimon-browser` V3 初回リリース  
位置づけ: 実装前の最終判断を固定し、AIエージェント・実装者・レビュー担当の共通前提とするための文書

---

## 1. この文書の目的

本書は、タビモン V3 に関する複数資料のうち、**初回リリースに必要な判断だけを固定するための意思決定シート**である。

V3提案書や統合仕様書には中長期の拡張案も含まれているが、初回リリースではスコープを絞らないと実装肥大・手戻り・設計ぶれが起きやすい。  
そのため本書では、**初回V3の正本**として以下を確定する。

- 初回V3に含める機能範囲
- 含めない機能範囲
- データモデルの最終方針
- V2 → V3 のマイグレーション方針
- 画面・遷移の最終方針
- 実装優先順位
- MVP完了条件

---

## 2. 初回V3の基本方針

### 2-1. V3のテーマ

**相棒との関係が深くなるアップデート**

### 2-2. 初回V3で実現すべきこと

初回V3は、新しい機能を大量に増やす版ではない。  
V2で成立したスマホ1画面のわかりやすい体験を維持しつつ、以下の3軸を追加して、毎回の冒険に「ちゃんと進んだ感」を与えることを目的とする。

- EXP
- きずな
- スキル熟練

### 2-3. 初回V3の設計原則

1. V2の美点を壊さない
2. プレイヤー向けUIはシンプルに保つ
3. 初回は軽量実装で成立を優先する
4. 拡張余地は残すが、本実装は最小限に留める
5. 演出より先にデータ反映と整合性を固める

## 4. 初回V3の確定判断一覧

## 4-1. スコープ判断

| 論点 | 確定判断 |
|---|---|
| 初回V3の範囲 | EXP / きずな / 熟練 / ステージ分離 / リザルト強化 / 軽量進化演出 に限定 |
| 分岐進化 | 初回では入れない。将来拡張用の受け皿のみ許容 |
| ホームのリアクション | 軽量版のみ。表情差分 + 吹き出し + 小エフェクトまで |
| 図鑑改修 | 初回では入れない |

| 進化専用画面 | 新設しない。リザルト内またはモーダルで完結 |

---

## 4-2. データモデル判断

| 論点 | 確定判断 |
|---|---|
| `OwnedMonsterV3` | 追加する |
| `bondRank` | 保存してよい。ただし基準値から再計算可能にする |
| 熟練データ | `useCount` 主体で持つ。stageは導出または補助保存 |
| 分岐進化の記録 | 初回では不要。必要なら `evolutionBranchId?: string` の受け皿のみ |
| きずな演出既読 | 最小限のみ。段階アップ既読の保持のみ許容 |
| `StageMasterV3.stageType` | `STORY` / `FARM` で固定 |
| `farmCategory` | `EXP` / `BOND` / `SKILL` で固定 |
| `difficultyTier` | `EARLY` / `LATE` を持たせる |
| `estimatedMinutes` | マスタ保持する |
| `DailyRecord` | 初回は `homeTapCount` を必須とする |
| 日付判定 | 端末ローカル日付基準 |

---

## 4-3. マイグレーション判断

| 論点 | 確定判断 |
|---|---|
| 既存モンスターのきずな初期値 | 全員 0 開始 |
| 既に進化済み個体 | そのまま維持 |
| 熟練初期値 | 全スキル 0 開始 |
| 図鑑進行 | 既存進行を引き継ぐ |
| 移行失敗時 | 旧データ退避 + ロールバック可能にする |

---

## 4-4. 画面遷移判断

| 論点 | 確定判断 |
|---|---|
| ホームから詳細 | 相棒カード全体で仲間詳細へ遷移 |
| ステージ選択 | `/adventure/stages` 内で 2 タブ切替 |
| リザルト後 | 候補あり → `/adventure/candidate`、候補なし → `/home` |
| 進化表示 | リザルト内フローまたはモーダルで完結 |
| 新規ルート追加 | 初回では原則増やさない |

## 5. 初回V3データモデル確定版

## 5-1. OwnedMonsterV3

```ts
interface OwnedMonsterV3 extends OwnedMonster {
  currentExp: number;
  bondPoints: number;
  bondRank: 0 | 1 | 2 | 3 | 4;
  skillProficiency: Record<string, {
    useCount: number;
    stage: 0 | 1 | 2 | 3;
  }>;
  evolutionBranchId?: string | null;
  bondMilestoneRead?: number[];
}
```

### 補足

- `bondRank` は保存してよいが、常に `bondPoints` から再計算できること
- `skillProficiency` の主データは `useCount`
- `stage` はUI表示・検索用の補助で持ってもよい
- `evolutionBranchId` は初回未使用でも可

---

## 5-2. StageMasterV3

```ts
interface StageMasterV3 extends StageMaster {
  stageType: 'STORY' | 'FARM';
  farmCategory?: 'EXP' | 'BOND' | 'SKILL';
  difficultyTier?: 'EARLY' | 'LATE';
  recommendedLevel?: number;
  estimatedMinutes: number;
  firstClearBonusExp?: number;
}
```

### 補足

- `STORY` は既存の進行用ステージ
- `FARM` は育成用周回ステージ
- 初回クリアボーナスは STORY のみ基本適用

---

## 5-3. DailyRecord

```ts
interface DailyRecord {
  date: string; // YYYY-MM-DD (local)
  homeTapCount: number;
}
```

### 補足

- 初回は最小責務のみ
- 将来のログインボーナス等はまだ持たせない

## 6. きずな・熟練・進化の確定ルール

## 6-1. きずな段階

初回V3では以下で固定する。

| Rank | 必要ポイント |
|---|---|
| 0 | 0 |
| 1 | 50 |
| 2 | 150 |
| 3 | 400 |
| 4 | 1000 |

## 6-2. ホームタップによるきずな

- 1日3回まで `+1`
- 4回目以降は反応のみ、数値加算なし

## 6-3. 熟練段階

初回V3では以下の簡易3段階とする。

| Stage | useCount目安 | 表示 |
|---|---:|---|
| 0 | 0 | まだなれていない |
| 1 | 5 | すこしなれてきた |
| 2 | 15 | かなりなれてきた |
| 3 | 30 | じゅくれん！ |

## 6-4. 進化

- 既存の Lv15 進化を維持
- 初回V3では単一路線のみ
- 進化専用ページは作らない
- リザルト中で気持ちよく見せる軽量演出のみ実装

## 9. 初回V3 MVP完了条件

以下を満たしたら、初回V3は「成立」と見なす。

### 必須条件

- ホームで相棒 + EXP + きずなが見える

- 進化が短く気持ちよく見える
- 既存セーブデータから安全に移行できる
- ビルドと主要導線の動作確認が通る
