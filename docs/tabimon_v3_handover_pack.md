# タビモン V3 別チャット引継ぎ資料

作成日: 2026-03-28

## 1. この資料の目的

この資料は、タビモン V3 の企画・UI/演出詳細設計・実装前論点整理までの経緯を、別チャットの後任AIが短時間で把握できるようにするための引継ぎ用サマリーである。

この資料だけで、以下を把握できるように整理している。

- 何を目指しているプロジェクトか
- V2 / V3 で何が決まっているか
- 5本の詳細仕様で何を確定したか
- まだ実装前に決める必要があることは何か
- 次チャットで何から着手すべきか
- 後任チャットに渡すべき関連資料は何か

---

## 2. プロジェクト概要

### 2-1. タイトル
タビモン（スマートフォン向けブラウザゲーム）

### 2-2. 現在地
- V1: プロトタイプとして成立済み
- V2: 世界観を維持したまま、モダンで洗練されたUIへ刷新する正式版として整理済み
- V3: UI刷新を土台に、相棒との関係性・育成体験・進化の山場を強化する拡張版を設計中

### 2-3. V3のテーマ
V3のテーマは、単なる機能追加ではなく、**「相棒との関係が深くなる体験」** をプレイヤーに感じさせること。

中核は以下の3軸。
- EXP
- きずな
- スキル熟練

加えて、
- ホームで相棒が主役に見えること
- ものがたり / そだてる の目的分岐
- リザルトでの報酬表示強化
- 進化を山場として見せること

を軸に設計が進んでいる。

## 3. ここまでの大きな結論

### 3-1. 役割分担
5本の仕様を横断すると、各画面の役割は次のように整理されている。

- ホーム = つながり
- 仲間詳細 = 理解
- ステージ選択 = 目的分岐
- リザルト = 報酬表示
- 進化 = 山場

この役割分担はかなり整合しており、大きな破綻はない。

### 3-2. 実装順の考え方
演出から入るのではなく、まずデータ基盤とリザルト反映を固めるべき、という整理になっている。

推奨優先順位は以下。

#### 優先度S
- `OwnedMonsterV3`
- `StageMasterV3`
- `DailyRecord`
- リザルトの EXP / きずな / 熟練 / ドロップ反映

#### 優先度A
- ホーム画面改修
- 仲間詳細画面改修
- ステージ選択画面改修

#### 優先度B
- きずなの軽量演出
- 単一路線の進化演出

#### 優先度C
- 本格アイドルアニメ
- 分岐進化の全面対応
- 大量の表情差分
- BGM / SE / PWA強化

### 3-3. 初回V3のMVP判断基準
最低限、初回V3で成立とみなす条件は以下。

- ホームで相棒 + EXP + きずなが見える
- 仲間詳細で EXP / きずな / 熟練が確認できる
- ステージ選択で `ものがたり / そだてる` が分かれる
- リザルトできずな・熟練・進化通知が出る
- 進化が短く気持ちよく見える

## 4. 詳細化済みの5本の仕様

以下の5テーマは、すでに詳細仕様としてかなり詰まっている。

### 4-1. ホーム画面改修仕様
目的:
- 相棒が主役に見える
- EXPときずなの成長を要約表示する
- タップリアクションで愛着を作る
- 主CTAは `ぼうけんへ しゅっぱつ`

重要ポイント:
- ホームは要約画面であり、詳細は仲間詳細へ逃がす
- 相棒カード領域全体から仲間詳細に遷移
- EXPは簡易バー
- きずなは段階 + 簡易バー
- ホームタップは1日3回まで +1、それ以降は反応のみ

### 4-2. 仲間詳細画面改修仕様
目的:
- EXP / きずな / 熟練の3軸をちゃんと理解できる
- 「うちの子感」を保ったまま、詳細確認できる

重要ポイント:
- 上部はビジュアル重視
- 中段で EXP / きずな / 熟練 を別文法で見せる
- EXPはフルバー + 数値
- きずなは段階 + バー + 残量
- 熟練はスキルごとのカード表示
- 相棒設定・手放しなど既存責務は維持

### 4-3. ステージ選択画面改修仕様
目的:
- `ものがたり` と `そだてる` をUIで意味ある分離にする

重要ポイント:
- `ものがたり` = 世界観 / 初回攻略 / 初回報酬 / 進行度
- `そだてる` = 主報酬 / 推奨Lv / 目安時間 / 用途
- 2タブに分けるだけでなく、カードの見せ方も変える

### 4-4. きずなの演出仕様
目的:
- きずなを内部数値ではなく、感情の積み上がりとして見せる

重要ポイント:
- 段階: 50 / 150 / 400 / 1000 pt
- 主報酬は数値ではなく、セリフ・表情・反応
- 場面ごとに演出を切り分ける
  - ホームタップ
  - 冒険開始
  - リザルト通常
  - リザルト失敗
  - 進化後
- 初回は量より規則性重視

### 4-5. 進化演出仕様
目的:
- 既存の Lv15進化を、V3では報酬体験の山場として見せ直す

重要ポイント:
- フローは `進化可能通知 → 進化確認 → （必要なら）分岐選択 → 演出 → 完了表示`
- 分岐は最大2択まで
- 比較項目は最大3つまで
- 演出は短く、スキップ可能
- 進化後に Before / After と変化内容を見せる
- 進化完了で `きずな +30`

## 5. まだ実装前に固める必要がある論点

ここは、AIエージェントに渡す前に判断が必要な未確定論点。

### 5-1. 初回V3の最終スコープ
決める必要があること:
- 分岐進化を初回V3に入れるか
- ホームの本格タップリアクションをどこまでやるか
- 設定画面 / BGM / SE / PWA強化を今回含めるか
- 図鑑改修を今回含めるか
- 進化用の独立URLを作るか

推奨:
- 初回V3は軽量版に寄せる
- 分岐進化は受け皿のみ、初回は単一路線優先
- 音やPWAは後回し

### 5-2. データモデルの最終確定
決める必要があること:
- `bondRank` を永続化するか都度計算にするか
- 熟練を uses で持つか stage だけで持つか
- 将来の分岐進化のための履歴保持をどうするか
- `DailyRecord` に何を持たせるか

推奨:
- `bondRank` は永続化よりも都度計算でもよいが、表示頻度次第で判断
- 熟練は uses 保持推奨（将来の調整余地が大きい）
- `DailyRecord` は初回は最小限でよい

### 5-3. V2 → V3 マイグレーション
決める必要があること:
- 既存個体のきずな初期値
- 既進化個体を維持するか巻き戻すか
- 熟練初期値
- 図鑑引継ぎ

推奨:
- 既進化個体は維持
- 熟練は0スタートでもよい
- きずなは一律0か、軽い推定付与かを要判断

### 5-4. 数値の固定値 / 可変値
決める必要があること:
- `BALANCE_V3` をどこまで唯一の真実の源にするか
- きずな段階値を固定するか
- 熟練段階値を固定するか
- `そだてる` の EXP をロックするか

推奨:
- 設定値として切り出しておく
- コード直書きは避ける

### 5-5. 軽量版 / 本格版の境界
決める必要があること:
- ホーム演出の上限
- きずなセリフの粒度
- 進化演出の上限
- 画像差分の必要数

推奨:
- 初回は軽量版固定
- 派手さではなく、短く分かりやすく嬉しい体験を優先

### 5-6. URL / 遷移設計
決める必要があること:
- `/evolution` を新設するか
- `ものがたり / そだてる` をURLに持つか
- `/settings` を今回出すか

推奨:
- 進化は初回はリザルト内フローでもよい
- タブ状態はまずUI state管理でよい

### 5-7. アセット要件
決める必要があること:
- 進化演出用画像の新規要否
- ホーム表情差分の必要数
- 熟練見た目変化の表現方法
- ステージタブ差別化のための新規装飾要素

推奨:
- 初回V3で本当に必要な最小セットに絞る
- 仕様先行で増やしすぎない

## 6. 次チャットで最初にやるべきこと

次チャットでは、新しいアイデア出しより、**未確定論点の意思決定** を優先するのが正しい。

推奨着手順は以下。

1. 初回V3の最終スコープを確定
2. データモデルの最終版を確定
3. V2 → V3 マイグレーション方針を確定
4. 進化フローのURL / 画面設計を確定
5. 軽量版演出の上限を確定
6. その後、AIエージェント向け実装指示書に変換

## 9. 注意事項

- いま必要なのは新規アイデア追加ではなく、未確定論点の意思決定
- 仕様はかなり強くなっているので、実装中に判断が揺れないようにすることが重要
- 後任チャットでは、まず資料整理ではなく「何を決めるか」の優先順位付けから始めた方がよい

## 10. 最終メモ

現状は「設計が弱い」のではなく、**設計が強くなってきたからこそ、実装前の最終判断が必要な段階** まで来ている。

大きな方向性はかなり良い。
次にやるべきことは、

- 境界を決める
- データを決める
- 移行を決める
- そのうえで実装指示に落とす

この順番を守ること。
